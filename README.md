## 📝 기능 요구 사항

### 1. 비상 근무일 배정 규칙
- [ ] 비상 근무일 배정
- [ ] 기본적으로 순번에 따라 배정
- [ ] 평일과 휴일 순번의 순서는 다를 수 있다
- [ ] 평일 순번, 휴일 순번에 각각 1회만 편성 되어야한다
- [ ] 연속 2일 근무 불가
  - 다음 근무자와 순서를 바꾼다 ex) 금토인 경우 토요일을 다음 휴일 근무자와 변경
  - 앞의 날짜부터 순서 변경
- [ ] 최소 5명, 최대 35명의 근무자 유지

### 2. 근무자 닉네임
- [ ] 중복 불가
- [ ] 최대 5자

### 3. 입출력 요구사항
- [ ] 평일이면서 법정공휴일의 경우에만 요일 뒤에 (휴일) 표기
- [ ] 평일 순번 또는 휴일 순번의 입력 값이 올바르지 않은 경우, '평일 순번'부터 다시 입력 받는다.
- [ ] 비상 근무를 배정할 월과 시작 요일의 입력 값이 올바르지 않은 경우, '비상 근무를 배정할 월과 시작 요일'부터 다시 입력받는다.

### 4. 기본 예외 처리
- [ ] 잘못된 입력 시
  - `IllegalArgumentException`을 발생시킨다.
  - `[ERROR]`로 시작하는 에러 메시지를 출력 후 특정 부분부터 입력을 다시 받는다.
- [ ] 모든 에러 메세지는 `[ERROR]`로 시작하는 에러 메시지를 출력한다. 이때 `Exception`이 아닌 `IllegalArgumentException`, `IllegalStateException` 등과 같은 명확한 유형을 처리한다.

## 🛠️ 프로그래밍 요구 사항

### 실행 환경
- JDK 버전: 21
- 프로그램 시작점: `Application.main()`
- 빌드 도구: Gradle

### 제약 사항
- `build.gradle` 파일 변경 금지
- 외부 라이브러리 추가 금지
- `System.exit()` 사용 금지
- 입력과 출력을 담당하는 클래스를 별도로 구현
- 사용자가 입력하는 값은 `camp.nextstep.edu.missionutils.Console`의 `readLine()`을 활용

### 코드 컨벤션
- 기본적으로 [Google Java Style Guide](https://google.github.io/styleguide/javaguide.html) 준수
- 추가로 [우아한테크코스 Java 코딩 컨벤션](https://github.com/woowacourse/woowacourse-docs/tree/main/styleguide/java) 준수
- `indent depth` ≤ 2
- `else` 및 `switch/case` 사용 금지
- 삼항 연산자 금지
- 메서드 길이는 15라인 이하
- 메서드는 한 가지 일만 수행
- 단위 테스트 작성
